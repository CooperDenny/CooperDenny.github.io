Project Overview

The Melbourne Pedestrian Counting System is a comprehensive Power BI dashboard designed to analyze pedestrian movement patterns in Melbourne. This project showcases advanced data visualization and analysis techniques to provide insights into pedestrian traffic trends.

Features:

  • Daily pedestrian counts
  • Average pedestrian counts per hour
  • Data differentiated by day of the week, weekday/weekend, and year
  • Slicer to filter data from almost 100 sensors
  • Mapped sensors using longitude and latitude coordinates

Data Source:

The data was sourced from pedestrian.melbourne.vic.gov.au, which provides hourly pedestrian counts from numerous sensors since January 2010.

Data Preparation:

To prepare the data for analysis, I utilized R for data cleaning, which involved:

  • Binding multiple CSV files
  • Fixing inconsistent date formats
  • Handling null and negative values
  • Pivoting the data from wide to long form

This cleaned data formed the foundation for my analysis in Power BI, enabling the creation of new variables and measures such as 4-week and 52-week averages.

Dashboard:

The dashboard consists of two main pages:

  • Daily Pedestrian Counts: Focuses on displaying the total pedestrian counts on a daily basis.
  • Average Pedestrian Counts Per Hour: Provides insights into the average pedestrian traffic per hour, categorized by day of the week, weekday/weekend, and year.
